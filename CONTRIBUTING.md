# Contributing

Thanks for checking out the repository.

## Branching

- Open pull requests against `develop`
- `main` is reserved for release-ready code
- Pull requests to `main` may be closed unless explicitly requested by the maintainer

## Before opening a PR

1. Sync `develop`
2. Create a feature or fix branch
3. Open the PR with base `develop`

## Notes

- CI currently relies on a self-hosted runner
- External forks may not reproduce the same build environment
